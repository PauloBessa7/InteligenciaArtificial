## 1. Conceitual – Importância da Preparação
**Pergunta:** Explique por que a etapa de preparação de dados pode impactar mais o desempenho de um modelo do que a escolha do algoritmo de Machine Learning.

**R:** A preparação do conjunto de dados é importante para garantir a consistência e o bom aprendizado que o algoritmo de ML terá sobre a base e, consequentemente, uma boa predição sobre a classe/atributo que se deseja prever. Caso a base não seja devidamente preparada, ela pode “puxar” para algum lado onde os dados são mais *outliers*, por exemplo.

**a) Quais são as possíveis estratégias para tratar esses valores?**

**b) Em quais situações remover registros é inadequado?**
**R:** Quando o objeto tem todos os outros atributos devidamente definidos.

    **R:** Usada quando a amostragem/conjunto possui dados normalmente distribuídos. É útil em modelos com meta-atributos numéricos e com grande variância nos dados do atributo, como salários de uma empresa.

## 4. Dados Categóricos
Você possui a variável: `clima = {sol, chuva, nublado}`

**a) Como transformar essa variável para uso em um modelo de Machine Learning?**
**R:** Podemos transformar a variável categórica usando técnicas como *One-Hot Encoding*, onde cada categoria vira uma coluna binária (ex: sol = [1,0,0], chuva = [0,1,0], nublado = [0,0,1]).

**b) Qual problema pode ocorrer se utilizarmos apenas codificação numérica simples (0, 1, 2)?**
**R:** Com atributos unicamente numéricos, podemos ter problemas onde o modelo cria relações de superioridade entre os valores (como 2 > 1 > 0), dando prioridade nas predições de uma forma que não existe na realidade.

## 5. Overfitting e Separação de Dados
**Explique por que dividimos o dataset em:**

* **Treino:**
    **R:** Dados de treino são uma amostragem do conjunto onde o modelo vai aprender a prever o valor do meta-atributo.
* **Validação:**
    **R:** Etapa onde avaliamos se as respostas de treino tiveram bons resultados.
* **Teste:**
    **R:** Com o modelo “treinado”, aplicamos ele em uma amostragem real e diferente do treino para, assim, validar com clareza sua performance.

**O que acontece se utilizarmos o conjunto de teste durante o treinamento?**
**R:** Ele terá uma assertividade muito grande (acurácia), o que trará a falsa impressão de que o modelo está bem treinado. Na verdade, em uma amostragem real e diferente do aprendizado, ele terá uma assertividade muito menor do que o esperado.

## 6. Balanceamento de Classes
Considere um problema onde 95% das amostras pertencem à classe A e 5% pertencem à classe B.

**a) Qual problema pode ocorrer?**
**R:** Se usar a classe A como amostragem predominante, o modelo terá uma performance/eficiência computacional muito baixa em relação a uma amostragem balanceada. Caso use apenas a classe B, terá uma amostragem muito pequena que não conseguirá gerar uma predição boa sobre o conjunto real.

**b) Cite duas técnicas para tratar esse desbalanceamento.**
**R:** Podemos fazer a separação da amostragem de forma normal (ou seja, 50/50, sem critérios de separação) ou fazer por algum critério onde podemos incluir, por exemplo, a classe de um atributo.
